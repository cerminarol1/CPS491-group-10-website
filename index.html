Client: Riverside Research

Riverside Research is a nonprofit organization that has been completing cutting edge research since it was established in 1967. Their research covers many fields which include radar, intelligence & surveillance, THz Imaging, and of course, Artificial Intelligence & Machine Learning.

Project Description

There are many different LLMs available, and all have varying degrees of competency in different fields. In order to maximize accuracy and efficiency, as well as minimize waste and cost, Riverside Research wants a way to choose the best LLM based on their various coding proficiencies. This will allow for programmers to have one convenient tool that will effectively act as the best LLM for each question, to make development in other projects faster.

Project Vision

To research and develop an effective routing tool for utilizing the strengths of multiple LLM models related to various software development processes and programming categories.

Project Goals

  1. Comprise and present research on various LLM models regarding their effectiveness in software development.
  2. Develop a functional web application that utilizes the varying strengths of multiple LLM models in software development.

Sprint 2

Sprint Goals

The goals for this sprint were concentrated around two main deliverables. These deliverables were a research paper that detailed categories for routing questions as well as the best LLMs for each category, and a working prototype that can demonstrate basic functionality.

Development Progress

In this sprint we engaged in a research project to determine the categories for routing questions to LLMs and determined the best LLMs for handling questions from each category. Our research consisted of consulting scholarly sources and developer resources for LLMs that fit within the acceptable criteria for the project, developing categories that adequately cover many programming problems that developers could face, finding benchmarks that uses accurate metrics to evaluate and LLM’s capabilities in a category, and evaluating each LLM’s aptitude for each category. The categories we decided on were Code Translation, Code Completion, Code Generation, Code Repair, Code Interpretation, Data Science, and Syntax Errors. We settled on three LLMs to use in the development cycle, each with their own distinct advantages in one or many categories. These LLMs are Llama3, Granite 3.0, and Codestral Mamba. Llama3 was found to handle questions on Code Translation, Code Completion, Code Generation, Code Repair, and Code Interpretation most effectively. Granite 3.0 was chosen to handle Data Science questions. Codestral Mamba was found to be most effective for Syntax Error questions.
